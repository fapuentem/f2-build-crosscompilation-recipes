This repository builds Quectel Connection Manager for a Jetson Orin NX
8GB running JetPack 5.1.2.

The build is started from an x86 Ubuntu workstation, but the actual
compile runs inside an ARM64 Ubuntu 20.04 Docker container through
Docker Buildx and QEMU. This keeps the resulting binaries compatible
with the Jetson user space and avoids linking against Ubuntu 24.04
libraries from the workstation.

This README is the single source of truth. The Dockerfile is generated
from the source block in this file using Org Babel tangling.

* Target

- Host machine: x86_64 Ubuntu 24.04
- Target device: Jetson Orin NX 8GB
- Target OS: JetPack 5.1.2 / Ubuntu 20.04 based root filesystem
- Target architecture: ARM64 / aarch64
- Main binary: ~quectel-CM~
- Optional proxy binaries: ~quectel-qmi-proxy~, ~quectel-mbim-proxy~, ~quectel-atc-proxy~, ~quectel-qrtr-proxy~
- Default install prefix on the Jetson: ~/usr/local/quectel-CM~

* Source tree sanity check

Run this before building. It catches missing source files early.

#+BEGIN_SRC bash :exports code :var SOURCE_DIR=quectel-source-dir()
  set -eu

  if [ ! -d "${SOURCE_DIR}" ]; then
    echo "missing source directory: ${SOURCE_DIR}"
    exit 1
  fi

  required_files="
  Makefile
  main.c
  QMIThread.h
  QmiWwanCM.c
  GobiNetCM.c
  QCQMUX.c
  QMIThread.c
  util.c
  qmap_bridge_mode.c
  mbim-cm.c
  device.c
  atc.c
  atchannel.c
  at_tok.c
  udhcpc.c
  quectel-qmi-proxy.c
  quectel-mbim-proxy.c
  quectel-atc-proxy.c
  default.script
  default.script_ip
  "

  missing=0
  for file in ${required_files}; do
    if [ ! -f "${SOURCE_DIR}/${file}" ]; then
      echo "missing: ${SOURCE_DIR}/${file}"
      missing=1
    fi
  done

  if [ "${missing}" -ne 0 ]; then
    echo
    echo "The Quectel source package is incomplete."
    echo "Get the missing files from the original Quectel CM release before building."
    exit 1
  fi
#+END_SRC

* Why Buildx is used

A normal Docker build on the workstation would build x86_64 binaries.
The Jetson needs ARM64 binaries.

Docker Buildx allows the workstation to run the build for another
architecture:

#+BEGIN_SRC bash :exports code
  docker buildx build --platform linux/arm64 ...
#+END_SRC

This recipe uses an ARM64 Ubuntu 20.04 container because JetPack 5.1.2
is based on Ubuntu 20.04.

* Build Quectel Connection Manager

Run this from the repository root on the x86 workstation.

    --platform linux/arm64 \
    --target artifact \

    --build-arg JOBS=4 \

* Install it on the Jetson

This section assumes the tarball is already copied to the Jetson at:

#+BEGIN_EXAMPLE
/tmp/quectel-CM-aarch64-ubuntu20.04.tar.xz
#+END_EXAMPLE

Therefore, extract it from ~/~ and let the archive place the files under
~/usr/local/quectel-CM~.

First, remove the broken nested installation:

#+BEGIN_SRC bash :exports code
  set -eu

  sudo pkill -INT quectel-CM 2>/dev/null || true
  sudo pkill -TERM quectel-CM 2>/dev/null || true

  sudo rm -f /usr/local/bin/quectel-CM
  sudo rm -f /usr/local/bin/quectel-mbim-proxy
  sudo rm -f /usr/local/bin/quectel-atc-proxy
  sudo rm -f /usr/local/bin/quectel-qmi-proxy

  sudo rm -rf /usr/local/quectel-CM
#+END_SRC

Inspect the tarball before extracting it:

#+BEGIN_SRC bash :exports code
  tar -tf /tmp/quectel-CM-aarch64-ubuntu20.04.tar.xz | head -n 30
#+END_SRC

If you see paths beginning with ~usr/local/quectel-CM/~, install it with this command:

#+BEGIN_SRC bash :exports code
  set -eu

  sudo tar -C / -xaf /tmp/quectel-CM-aarch64-ubuntu20.04.tar.xz

  sudo ln -sf /usr/local/quectel-CM/bin/quectel-CM /usr/local/bin/quectel-CM
  sudo ln -sf /usr/local/quectel-CM/bin/quectel-mbim-proxy /usr/local/bin/quectel-mbim-proxy
  sudo ln -sf /usr/local/quectel-CM/bin/quectel-atc-proxy /usr/local/bin/quectel-atc-proxy

  if [ -x /usr/local/quectel-CM/bin/quectel-qmi-proxy ]; then
    sudo ln -sf /usr/local/quectel-CM/bin/quectel-qmi-proxy /usr/local/bin/quectel-qmi-proxy
  fi
#+END_SRC

Verify the installation:

#+BEGIN_SRC bash :exports code
  which quectel-CM
  readlink -f "$(which quectel-CM)"
  ls -l /usr/local/quectel-CM/bin/
  file /usr/local/quectel-CM/bin/quectel-CM
#+END_SRC

Expected result:

#+BEGIN_EXAMPLE
/usr/local/bin/quectel-CM
/usr/local/quectel-CM/bin/quectel-CM
#+END_EXAMPLE

* Prepare the modem on the Jetson

Check that the Quectel modem is visible:

#+BEGIN_SRC bash :exports code
  lsusb | grep -i quectel || true
  ls -l /dev/cdc-wdm* 2>/dev/null || true
  ip link show | grep -E "wwan|usb" || true
#+END_SRC

For QMI mode, the common Jetson interface is ~wwan0~ with a control
channel such as ~/dev/cdc-wdm0~.

Stop services that may already control the modem before testing
~quectel-CM~ manually:

#+BEGIN_SRC bash :exports code
  sudo systemctl stop ModemManager 2>/dev/null || true
  sudo systemctl stop NetworkManager 2>/dev/null || true
  sudo pkill -INT quectel-CM 2>/dev/null || true
  sudo pkill -TERM quectel-CM 2>/dev/null || true
#+END_SRC

Replace the APN with the SIM provider APN.

#+NAME: sim-apn

* QMI diagnostic checks

These checks help confirm that the modem is registered before blaming
~quectel-CM~.

#+BEGIN_SRC bash :exports code :var TARGET_HOST=f2sc01-target() :var SSH_PASS=f2sc01-ssh-pass()
  sshpass -p "${SSH_PASS}" ssh "nvidia@${TARGET_HOST}" '
    sudo qmicli -d /dev/cdc-wdm0 --dms-get-operating-mode
    sudo qmicli -d /dev/cdc-wdm0 --uim-get-card-status
    sudo qmicli -d /dev/cdc-wdm0 --nas-get-serving-system
    sudo qmicli -d /dev/cdc-wdm0 --nas-get-signal-strength
  '
#+END_SRC

Expected signs of success:

- SIM card is present and ready.
- Operating mode is online.
- Serving system is registered.
- Signal strength is available.
- ~wwan0~ receives an IPv4 address or the CM log shows IPv4 settings.

* Optional systemd service

Create a service only after the manual test works.

#+NAME: quectel-cm-service
#+BEGIN_SRC systemd :tangle quectel-CM.service
  [Unit]
  Description=Quectel Connection Manager
  After=network-pre.target
  Wants=network-pre.target

  [Service]
  Type=simple
  ExecStart=/usr/local/bin/quectel-CM -s internet -i wwan0 -v -f /var/log/quectel-CM/quectel-CM.log
  Restart=always
  RestartSec=5

  [Install]
  WantedBy=multi-user.target
#+END_SRC

Copy and enable the service:

#+BEGIN_SRC bash :exports code :var TARGET_HOST=f2sc01-target() :var SSH_PASS=f2sc01-ssh-pass()
  sshpass -p "${SSH_PASS}" scp quectel-CM.service \
    "nvidia@${TARGET_HOST}:/tmp/quectel-CM.service"

  sshpass -p "${SSH_PASS}" ssh "nvidia@${TARGET_HOST}" '
    sudo mkdir -p /var/log/quectel-CM
    sudo mv /tmp/quectel-CM.service /etc/systemd/system/quectel-CM.service
    sudo systemctl daemon-reload
    sudo systemctl enable --now quectel-CM.service
    systemctl status quectel-CM.service --no-pager
  '
#+END_SRC

Check the service logs:

#+BEGIN_SRC bash :exports code :var TARGET_HOST=f2sc01-target() :var SSH_PASS=f2sc01-ssh-pass()
  sshpass -p "${SSH_PASS}" ssh "nvidia@${TARGET_HOST}" '
    journalctl -u quectel-CM.service -n 100 --no-pager
    sudo tail -n 100 /var/log/quectel-CM/quectel-CM.log
  '
#+END_SRC

* Troubleshooting

** Build fails with missing source files

Run the source sanity check. If ~mbim-cm.c~ or ~quectel-qmi-proxy.c~ is
missing, copy them from the original Quectel Connection Manager release
into the repository and rebuild.

** Build fails with warnings treated as errors

The Quectel Makefile uses ~-Werror~. For a temporary diagnostic build,
change the Docker build argument:

#+BEGIN_SRC bash :exports code
  docker buildx build \
    --platform linux/arm64 \
    --target artifact \
    --output type=local,dest=out \
    -f Dockerfile.quectel-cm-jetpack512 \
    --build-arg STRICT_WARNINGS=no \
    --build-arg SOURCE_DIR="Quectel_QConnectManager_Linux_V1.6.7-20251226-2" \
    .
#+END_SRC

Do not keep this relaxed build for production unless you have reviewed
the warnings.

** The modem is detected but no IP is assigned

Check raw IP mode and routes:

#+BEGIN_SRC bash :exports code :var TARGET_HOST=f2sc01-target() :var SSH_PASS=f2sc01-ssh-pass()
  sshpass -p "${SSH_PASS}" ssh "nvidia@${TARGET_HOST}" '
    cat /sys/class/net/wwan0/qmi/raw_ip 2>/dev/null || true
    ip addr show wwan0
    ip route show
    sudo tail -n 200 /var/log/quectel-CM/quectel-CM.log
  '
#+END_SRC

If Ethernet is still the preferred default route, add explicit route
metric handling in your network failover layer. Keep ~quectel-CM~ focused
on bringing up the modem and interface.

** ModemManager or NetworkManager conflicts with quectel-CM

Only one manager should control the modem during the test. Stop the
other managers while testing manually:

#+BEGIN_SRC bash :exports code
  sudo systemctl stop ModemManager NetworkManager
#+END_SRC

For production, decide whether the F2 device uses ~quectel-CM~ directly
or a higher-level manager such as ModemManager/NetworkManager. Avoid
running both against the same modem control port.

* Dockerfile source

This block is tangled into ~Dockerfile.quectel-cm-jetpack512~.

#+BEGIN_SRC dockerfile :tangle Dockerfile.quectel-cm-jetpack512
  # syntax=docker/dockerfile:1.7

  ARG UBUNTU_TAG=20.04
  FROM ubuntu:${UBUNTU_TAG} AS build

  ARG JOBS=4
  ARG STRICT_WARNINGS=yes
  ARG PREFIX=/usr/local/quectel-CM
  ARG SOURCE_DIR=Quectel_QConnectManager_Linux_V1.6.7-20251226-2

  ENV DEBIAN_FRONTEND=noninteractive
  SHELL ["/bin/bash", "-euxo", "pipefail", "-c"]

  RUN apt-get update && \
      apt-get install -y --no-install-recommends \
        ca-certificates \
        build-essential \
        make \
        file \
        xz-utils \
        pkg-config \
        linux-libc-dev && \
      rm -rf /var/lib/apt/lists/*

  WORKDIR /workspace
  COPY . /workspace

  RUN test -d "${SOURCE_DIR}" || \
        (echo "missing source directory: ${SOURCE_DIR}" >&2; exit 1)

  WORKDIR /workspace/${SOURCE_DIR}

  RUN required_files=( \
        Makefile \
        main.c \
        QMIThread.h \
        QmiWwanCM.c \
        GobiNetCM.c \
        QCQMUX.c \
        QMIThread.c \
        util.c \
        qmap_bridge_mode.c \
        mbim-cm.c \
        device.c \
        atc.c \
        atchannel.c \
        at_tok.c \
        udhcpc.c \
        quectel-qmi-proxy.c \
        quectel-mbim-proxy.c \
        quectel-atc-proxy.c \
        default.script \
        default.script_ip \
      ); \
      missing=0; \
      for file in "${required_files[@]}"; do \
        if [[ ! -f "${file}" ]]; then \
          echo "missing: ${SOURCE_DIR}/${file}" >&2; \
          missing=1; \
        fi; \
      done; \
      if [[ "${missing}" -ne 0 ]]; then \
        echo "Quectel CM source tree is incomplete." >&2; \
        exit 1; \
      fi

  RUN if [[ "${STRICT_WARNINGS}" == "no" ]]; then \
        sed -i 's/-Werror//g' Makefile; \
      fi; \
      make -j"${JOBS}" release

  RUN install -d \
        /out/rootfs"${PREFIX}"/bin \
        /out/rootfs"${PREFIX}"/scripts \
        /out/rootfs"${PREFIX}"/doc && \
      install -m 0755 quectel-CM /out/rootfs"${PREFIX}"/bin/quectel-CM && \
      for bin in quectel-qmi-proxy quectel-mbim-proxy quectel-atc-proxy quectel-qrtr-proxy; do \
        if [[ -x "${bin}" ]]; then \
          install -m 0755 "${bin}" /out/rootfs"${PREFIX}"/bin/"${bin}"; \
        fi; \
      done && \
      install -m 0755 default.script /out/rootfs"${PREFIX}"/scripts/default.script && \
      install -m 0755 default.script_ip /out/rootfs"${PREFIX}"/scripts/default.script_ip && \
      if [[ -f ReleaseNote.txt ]]; then \
        install -m 0644 ReleaseNote.txt /out/rootfs"${PREFIX}"/doc/ReleaseNote.txt; \
      fi && \
      mkdir -p /out/artifacts && \
      tar -C /out/rootfs -caf /out/artifacts/quectel-CM-aarch64-ubuntu20.04.tar.xz . && \
      file /out/rootfs"${PREFIX}"/bin/quectel-CM > /out/artifacts/quectel-CM-aarch64-ubuntu20.04.file.txt && \
      /out/rootfs"${PREFIX}"/bin/quectel-CM --help > /out/artifacts/quectel-CM-aarch64-ubuntu20.04.smoke.txt 2>&1 || true

  FROM scratch AS artifact
  COPY --from=build /out/artifacts/ /
#+END_SRC
